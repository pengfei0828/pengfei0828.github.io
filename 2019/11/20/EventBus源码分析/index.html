二、register 方法

方法的注释主要说的是：注册给定的订阅者来接收事件，当订阅者不再需要接受事件的时候应该调用 unregister(Object) 来取消注册。订阅者必须要有 Subscribe 注解的事件处理方法，Subscribe 注解允许我们来配置 ThreadMode 和 Priority 等信息。

直观上看这个方法主要做了两件事情：

  • 标注 ① 处，通过直译的方式可以猜到通过一个 SubscriberMethodFinder 中的 findSubscriberMethods 方法查询一个 SubscriberMethod 类型的集合。
  • 标注 ② 处，遍历 ① 处得到的集合通过 subscribe 方法具体进行注册。

这里有必要说一下 SubscriberMethod 类：

SubscriberMethod 类最主要的作用就是保存订阅者方法的信息，例如方法、线程、事件类型、优先级、是否为黏性事件等信息。

标注 ① 查找订阅方法 （SubscriberMethodFinder # findSubscriberMethods）

METHOD_CACHE 是以类名为 key 订阅者方法集合为 value 的 Map 对象，首先通过类名进行查找，若 subscriberMethods 不等于 null 则直接返回，若为 null 则根据布尔对象 ignoreGeneratedIndex（默认值是 false，当然你可以使用建造者模式构造 EventBus 对象时通过 EventBusBuilder.ignoreGeneratedIndex(boolean ignoreGeneratedIndex) 来改变该值） 的值来选择查询方法，ignoreGeneratedIndex 属性表示是否忽略注解器生成的 MyEventBusIndex。找到订阅方法后又做了一层判断，若 subscriberMethods 为空（元素个数为0，与上面的 null 有区别），则会抛出一个异常，这个异常主要指的是在当前类以及当前类的父类中没有找到任何一个带有 @Subscribe 注解并且被 public 修饰的方法，否则会通过 METHOD_CACHE 存储当前的订阅方法集合，最后返回订阅方法。

FindState 类

通过 initForSubscriber 方法初始化 subscriberClass、clazz、skipSuperClasses（是否指向父类）、subscriberInfo 字段，通过 recycle 方法回收各对象，moveToSuperclass 方法主要是向上查找，若 skipSuperClasses = true 或查找到的父类为系统类，则 clazz = null，否则一直向上查找。

findUsingReflectionInSingleClass 方法

findUsingReflectionInSingleClass 方法从整体上看做了两个操作：

  • 获取 Method 方法数组
  • 遍历 Method 方法数组

1.如何获取 Method 方法数组？

这里采用 getDeclaredMethods方法 Java 反射来获取，源码中有一段注释，主要讲的是 getDeclaredMethods 方法比 getMethods 方法更高效，特别是在像 Activitie 中。

2.遍历 Method 数组

通过 Method 的 getModifiers 方法获取方法的修饰符，必须被 public 修饰且不被 static 或 abstract 修饰的方法是符合条件的，否则会在 strictMethodVerification （默认值是 false，当然你可以使用建造者模式构造 EventBus 对象时通过 EventBusBuilder.strictMethodVerification(boolean strictMethodVerification) 来改变该值）和 isAnnotationPresent(Subscribe.class) 同为 true 时，抛出异常。

A.isAnnotationPresent(B.class) 可解释为：注释 B 是否作用在 A 上，在上面的主要作用是想判断该方法有没有 @Subscribe 注解。

继续看对符合条件的方法做了什么操作，通过 getParameterTypes 方法获取方法的参数，若参数的个数大于 1，会抛出这里只允许一个参数的异常。若参数个数为 1，接着判断方法中是否包含 @Subscribe 注解，若包含该注解，会调用 findState.checkAdd(method, eventType) 方法再次去验证。当以上所有的条件都满足了之后，最终会通过 SubscriberMethod 中的构造函数创建 SubscriberMethod 对象并保存至 FindState 里面的集合 subscriberMethods 中。
